Naïve Bayes

The Naïve Bayes Classifier extension fits the Naïve Bayes classification model for a categorical dependent variable. You can specify whether all available predictors are used or whether the procedure selects the best predictors. The extension can also be used for predictor selection without classification.

Variables

Dependent Variable. Specify the numeric or string variable to analyze.

Variable Specification Method. You can specify either a set of variables to exclude from the analysis or the set of factors and covariates to include in the analysis. If you don't specify any variables to exclude or any candidate factors or covariates, then all variables (except the dependent variable and the weight variable) are included.

Variables to Exclude. Specify variables to exclude from the analysis. By default, the procedure selects the best predictors from the remaining variables (the dependent variable and the weight variable are always excluded).

Factors and Covariates. Specify the factors and covariates to include in the analysis. Factors are treated as categorical variables and can have numeric values or string values. Covariates are treated as scale variables and must be numeric. By default, the procedure selects the best predictors from the specified set of factors and covariates.

Forced Variables. You can specify factors and covariates that are always used in the classification model. Forced factors are treated as categorical variables and can have numeric values or string values. Forced covariates are treated as scale variables and must be numeric.

Output

Case processing summary. Summarizes the number of cases that are included and excluded in the analysis.

Excluded predictors. Lists predictors that are excluded due to missing or constant values for all cases. The excluded predictors are displayed by type (categorical or scale).

Selected predictors. Displays selected predictors by type (categorical or scale).

Summary of predictor subsets. Displays a summary of the sequence of predictor subsets.

Classification of cases. Displays the number of cases that are classified correctly and incorrectly for each dependent variable category. If test data are defined, classification results are given for the training and the test samples. If test data are not defined, classification results are given only for the training sample.

Save

Predicted Value and Predicted Probabilities. You can save predicted values and predicted probabilities as new variables in the active dataset. The predicted value is the dependent variable category with the highest posterior probability as estimated by the Naïve Bayes model. The predicted probabilities of the first N categories of the dependent variable are saved, where N is 25 by default.

Options

User-Missing Values for Categorical Variables. Controls whether user-missing values for categorical variables are excluded or included in the analysis. User-missing values for scale variables are always excluded. System-missing values for any variables are always excluded.

Training Sample. You can partition the active dataset into training and test samples by specifying a percentage of randomly selected cases to assign to the training sample.

Predictor Selection.

• Select best subset from sequence of subsets. By default, the Naïve Bayes procedure selects the best subset of predictors from a sequence of subsets, with an increasing number of predictors in each subset. The procedure automatically determines the size of the largest subset but you can also specify an explicit maximum subset size. When the active dataset is partitioned into training and test samples, you can choose whether the best subset is determined from a pseudo-BIC criterion that is based on the training sample or a test data criterion that is based on the test sample.

• Select best subset of specified size. This choice specifies the exact number of predictors to be included in the best subset.

• Use all specified predictors. This choice indicates that all specified predictors are included in the final subset and is useful for model building without predictor selection.

Time limit. Specifies the maximum number of minutes during which the Naïve Bayes procedure can be run. If the time limit is exceeded, the procedure is terminated and no results are given. Specifying a value of 0 turns off the timer.

Maximum memory for storing training data. Specifies the maximum amount of memory in megabytes (MB) that the Naïve Bayes procedure uses to store training data records when computing the average log-likelihood. If the amount of memory that is required to store records is larger, simulated data are used instead.

Number of bins for scale predictors. Specifies the number of bins to use when dividing the domain of a scale predictor into equal-width bins.

Additional Features

This dialog generates syntax for the NAIVEBAYES command. In NAIVEBAYES syntax, you can specify a variable that partitions the dataset into training and test samples. Also, you can save the Naïve Bayes model to an XML file so that you can score new data with the model.

For more information about the Naïve Bayes procedure, see the documentation for the NAIVEBAYES command in the IBM® SPSS® Statistics Help system.
