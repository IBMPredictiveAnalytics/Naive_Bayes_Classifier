Naiver Bayes-Klassifikator

Die Naive Bayes-Klassifikator-Erweiterung passt das Naive Bayes-Klassifikationsmodell für eine kategoriale abhängige Variable an. Sie können festlegen, ob alle verfügbaren Prädiktoren verwendet werden oder ob die Prozedur die besten Prädiktoren auswählt. Die Erweiterung kann auch für die Auswahl von Prädiktoren ohne Klassifikation verwendet werden.

Variablen

Abhängige Variable. Geben Sie die zu analysierende numerische oder String-Variable an.

Variablenspezifikationsmethode. Sie können entweder eine Gruppe von Variablen angeben, die von der Analyse ausgeschlossen werden sollen, oder die Gruppe von Faktoren und Kovariaten, die in die Analyse einbezogen werden sollen. Wenn Sie keine auszuschließenden Variablen oder mögliche Faktoren oder Kovariaten angeben, werden alle Variablen (außer der abhängigen Variable und der Gewichtungsvariable) einbezogen.

Auszuschließende Variablen. Geben Sie die Variablen an, die von der Analyse ausgeschlossen werden sollen. Standardmäßig wählt die Prozedur die besten Prädiktoren aus den verbleibenden Variablen aus (die abhängige Variable und die Gewichtungsvariable werden immer ausgeschlossen).

Faktoren und Kovariaten. Geben Sie die Faktoren und Kovariaten an, die in die Analyse einbezogen werden sollen. Faktoren werden als kategoriale Variablen behandelt und können numerische Werte oder String-Werte haben. Kovariaten werden als Skalenvariablen behandelt und müssen numerisch sein. Standardmäßig wählt die Prozedur die besten Prädiktoren aus der angegebenen Gruppe von Faktoren und Kovariaten aus.

Zwangsvariablen. Sie können Faktoren und Kovariaten angeben, die immer im Klassifikationsmodell verwendet werden. Erzwungene Faktoren werden als kategoriale Variablen behandelt und können numerische Werte oder String-Werte haben. Erzwungene Kovariaten werden als Skalenvariablen behandelt und müssen numerisch sein.

Ausgabe

Zusammenfassung der Fallverarbeitung. Fasst die Fälle zusammen, die in die Analyse aufgenommen bzw. aus ihr ausgeschlossen werden.

Ausgeschlossene Prädiktoren. Listet Prädiktoren auf, die aufgrund fehlender oder konstanter Werte für alle Fälle ausgeschlossen sind. Die ausgeschlossenen Prädiktoren werden nach Typ (kategorial oder Skala) angezeigt.

Ausgewählte Prädiktoren. Zeigt ausgewählte Prädiktoren nach Typ (kategorial oder Skala) an.

Zusammenfassung der Prädiktorsubsets. Zeigt eine Zusammenfassung der Folge der Prädiktorsubsets an.

Klassifikation von Fällen. Zeigt die Anzahl der Fälle an, die für jede Kategorie der abhängigen Variablen richtig und falsch klassifiziert wurden. Wenn Testdaten definiert sind, werden die Klassifikationsergebnisse für die Trainings- und die Teststichprobe angegeben. Wenn keine Testdaten definiert sind, werden die Klassifikationsergebnisse nur für die Trainingsstichprobe angegeben.

Speichern

Vorhergesagter Wert und vorhergesagte Wahrscheinlichkeiten. Sie können vorhergesagte Werte und vorhergesagte Wahrscheinlichkeiten als neue Variablen im aktiven Dataset speichern. Der vorhergesagte Wert ist die Kategorie der abhängigen Variable mit der höchsten A-posteriori-Wahrscheinlichkeit, die durch das Naive Bayes-Modell geschätzt wird. Die vorhergesagten Wahrscheinlichkeiten der ersten N Kategorien der abhängigen Variable werden gespeichert, wobei N standardmäßig 25 ist.

Optionen

Benutzerdefiniert fehlende Werte für kategoriale Variablen. Steuert, ob benutzerdefiniert fehlende Werte für kategoriale Variablen in die Analyse aufgenommen bzw. aus ihr ausgeschlossen werden. Benutzerdefiniert fehlende Werte für metrische Variablen werden stets ausgeschlossen. Systemdefiniert fehlende Werte für beliebige Variablen werden immer ausgeschlossen.

Trainingsstichprobe. Sie können das aktiven Dataset in eine Trainings- und eine Teststichprobe aufteilen, indem Sie einen Prozentsatz zufällig ausgewählter Fälle angeben, der der Trainingsstichprobe zugeordnet werden soll.

Prädiktorauswahl.

• Bestes Subset aus einer Folge von Subsets auswählen. Standardmäßig wählt die Naive Bayes-Prozedur die besten Subsets von Prädiktoren aus einer Reihe von Subsets aus, wobei die Anzahl der Prädiktoren in jedem Subset zunimmt. Die Prozedur bestimmt automatisch die Größe des größten Subsets, aber Sie können auch explizit eine maximale Subsetgröße angeben. Wenn das aktive Dataset in Trainings- und Teststichproben unterteilt ist, können Sie wählen, ob das beste Subset anhand eines Pseudo-BIC-Kriteriums, das auf der Trainingsstichprobe basiert, oder eines Testdatenkriteriums, das auf der Teststichprobe basiert, bestimmt wird.

• Bestes Subset der angegebenen Größe auswählen. Diese Auswahl legt die genaue Anzahl der Prädiktoren fest, die in das beste Subset aufgenommen werden.

• Alle angegebenen Prädiktoren verwenden. Diese Auswahl gibt an, dass alle angegebenen Prädiktoren im endgültigen Subset enthalten sind, und ist für die Modellerstellung ohne Prädiktorauswahl nützlich.

Zeitlimit. Legt die maximale Dauer (in Minuten) fest, während der die Naive Bayes-Prozedur ausgeführt werden kann. Wird das Zeitlimit überschritten, wird die Prozedur abgebrochen und es werden keine Ergebnisse angezeigt. Bei einem Wert von 0 wird der Zeitgeber ausgeschaltet.

Maximale Speichergröße für das Speichern von Trainingsdaten. Gibt die maximale Speichermenge in Megabyte (MB) an, die die Naive Bayes-Prozedur zum Speichern von Trainingsdatensätzen bei der Berechnung der durchschnittlichen Log-Likelihood verwendet. Ist der für das Speichern von Datensätzen erforderliche Speicherplatz größer, werden stattdessen simulierte Daten verwendet.

Anzahl der Klassen für metrische Prädiktoren. Gibt die Anzahl der Klassen an, die bei der Unterteilung der Domäne eines metrischen Prädiktors in Klassen mit gleicher Breite verwendet werden sollen.

Weitere Funktionen

In diesem Dialogfeld wird Syntax für den Erweiterungsbefehl NAIVEBAYES erzeugt. In der Syntax von NAIVEBAYES können Sie eine Variable angeben, die das Dataset in Trainings- und Teststichproben unterteilt. Außerdem können Sie das Naive Bayes-Modell in einer XML-Datei speichern, um neue Daten mit dem Modell zu bewerten.

Weitere Informationen über die Naive Bayes-Prozedur finden Sie in der Dokumentation zum Befehl NAIVEBAYES im IBM SPSS Statistics-Hilfesystem.
