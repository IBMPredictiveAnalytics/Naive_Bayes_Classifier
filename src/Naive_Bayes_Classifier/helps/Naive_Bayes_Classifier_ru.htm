Наивный Байес

Расширение Классификатор Наивный Байес подгоняет модель наивной байесовской классификации для категориальной зависимой переменной. Можно указать, используются ли все доступные предикторы или же процедура выбирает лучшие предикторы. Это расширение можно также использовать для выбора предикторов без классификации.

Переменные

Зависимая переменная. Задайте числовую или строковую переменную для анализа.

Метод спецификации переменных. Можно задать либо набор переменных для исключения из анализа, либо набор факторов и ковариат для включения в анализ. Если вы не указываете никаких переменных для исключения и никаких факторов или ковариат-кандидатов, включаются все переменные (кроме зависимой переменной и переменной веса).

Переменные для исключения. Укажите переменные для исключения из анализа. По умолчанию процедура выбирает лучшие предикторы из оставшихся переменных (зависимая переменная и переменная веса всегда исключаются).

Факторы и ковариаты. Укажите факторы и ковариаты для включения в анализ. Факторы рассматриваются как категориальные переменные и могут иметь числовые или строковые значения. Ковариаты рассматриваются как количественные переменные и должны быть числовыми. По умолчанию процедура выбирает лучшие предикторы из заданного набора факторов и ковариат.

Принудительно используемые переменные. Можно задать факторы и ковариаты, всегда используемые в модели классификации. Принудительно используемые факторы рассматриваются как категориальные переменные и могут иметь числовые или строковые значения. Принудительно используемые ковариаты рассматриваются как количественные переменные и должны быть числовыми.

Вывод

Сводка обработки наблюдений. Суммирует число наблюдений, включенных в анализ и и исключенных из него.

Исключенные предикторы. Список предикторов, исключенных из-за пропущенных или постоянных для всех наблюдений значений. Исключенные предикторы выводятся по типам (категориальные или количественные).

Выбранные предикторы. Выводятся выбранные предикторы по типам (категориальные или количественные).

Сводка поднаборов предикторов. Выводит сводку последовательности поднаборов предикторов.

Классификация наблюдений. Число наблюдений, классифицированных правильно и неправильно, для каждой категории зависимых переменных. Если тестовые данные определены, то для обучающих и испытательных выборок приводятся результаты классификации. Если тестовые данные не определены, то результаты классификации даются только для обучающих выборок.

Сохранить

Предсказанные значения и предсказанные вероятности. Можно сохранить предсказанные значения и предсказанные вероятности в качестве новых переменных в активном наборе данных. Предсказанное значение - это категория зависимой переменной с самой высокой постериорной вероятностью, которая оценивается наивной байесовской моделью. Сохраняются предсказанные вероятности первых N категорий зависимой переменной, где N по умолчанию - 25.

Параметры

Пользовательские значения отсутствия для категориальных переменных. Управляет тем, будут ли пользовательские значения отсутствия для категориальных переменных исключаться из анализа или включаться в анализ. Пользовательские пропущенные значения для количественных переменных всегда исключаются. Системные пропущенные значения для любых переменных всегда исключаются.

Выборка обучения. Вы можете разделить активный набор данных на выборки обучения и тестирования, задав процент случайным образом выбранных наблюдений для назначения в выборку обучения.

Выбор предиктора.

• Выбрать наилучшее подмножество из последовательности подмножеств. По умолчанию наивная байесовская процедура выбирает наилучшее подмножество предикторов из последовательности подмножеств с нарастающим числом предикторов в каждом подмножестве. Эта процедура определяет размер наибольшего подмножества автоматически, но можно также задать максимальный размер подмножества явно. Когда активный набор данных разделяется на выборки обучения и тестирования, можно выбрать, определяется ли оптимальное подмножество по критерию псевдо-BIC на основе выборки обучения или по критерию данных тестирования на основе выборки тестирования.

•  Выбрать наилучшее подмножество указанного размера. В этом варианте задается точное число предикторов, включаемых в наилучшее подмножество.

• Использовать все указанные предикторы.. Этот выбор указывает, что все заданные предикторы включаются в окончательное подмножество; это полезно для построения модели без выбора предикторов.

Предельное время. Задает максимальное время в минутах, в течение которого может выполняться наивная байесовская процедура. Если предел времени превышен, процедура будет прервана без выдачи результатов. Если задать 0, то таймер будет выключен.

Максимальный объем памяти для хранения данных обучения. Задает максимальный объем памяти в мегабайтах, который наивная байесовская процедура использует для хранения записей данных обучения при вычислении среднего логарифма правдоподобия. Если объем памяти, необходимый для хранения записей, больше, вместо этого используются имитированные данные.

Число интервалов для количественных предикторов. Задает число интервалов, используемых при делении домена количественного предиктора на интервалы равной ширины.

Дополнительные возможности

Это диалоговое окно формирует синтаксис для команды NAIVEBAYES. В синтаксисе NAIVEBAYES можно задать переменную, по которой набор данных разбивается на выборки обучения и тестирования. Кроме того, вы можете сохранить наивную байесовскую модель в файл XML, чтобы можно было оценит новые данные по этой модели.

Дополнительную информацию о наивной байесовской процедуре смотрите в документации для команды NAIVEBAYES в справочной системе IBM SPSS Statistics.
